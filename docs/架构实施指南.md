## 📝 下一步实施步骤

### 步骤 1: 迁移 Domain 模型（优先级：高）

**目标**: 将 `SURS.App/Models` 中的模型迁移到 `SURS.Core/Domain`

**操作**:

1. **创建 Domain 文件夹结构**
   ```
   SURS.Core/Domain/
   ├── Entities/
   │   ├── Report/
   │   ├── Uterus/
   │   ├── Endometrium/
   │   ├── Adnexa/
   │   └── Fluid/
   ├── ValueObjects/
   └── Enums/
   ```

2. **按模块迁移模型**
   - 先迁移简单的实体（如 ORadsResult，已完成）
   - 再迁移复杂的聚合根（SursReport）

3. **更新命名空间**
   - 从 `SURS.App.Models` 改为 `SURS.Core.Domain.Entities`

4. **更新项目引用**
   - `SURS.App` 引用 `SURS.Core`
   - `SURS.Infrastructure` 引用 `SURS.Core`

---

### 步骤 2: 迁移业务逻辑（优先级：高）

**目标**: 将 `ORadsCalculator` 迁移到 `SURS.Core/Business`

**操作**:

1. **复制 ORadsCalculator.cs 到 SURS.Core/Business/**
2. **实现 IORadsCalculator 接口**
3. **更新命名空间和引用**
4. **从 SURS.App 中移除原文件**

---

### 步骤 3: 实现基础设施服务（优先级：高）

**目标**: 在 `SURS.Infrastructure` 中实现服务接口

**操作**:

1. **实现 PdfService**
   - 从 `SURS.App/Services/PdfService.cs` 复制
   - 实现 `IPdfGenerator` 接口
   - 更新命名空间

2. **创建 FileReportRepository**
   - 实现 `IReportRepository` 接口
   - 使用 JSON 序列化保存/加载报告

3. **创建 ConfigurationService**
   - 读取 appsettings.json
   - 管理用户配置

4. **创建 FileLogger**
   - 实现 `ILogger` 接口
   - 写入日志文件

---

### 步骤 4: 引入依赖注入（优先级：中）

**目标**: 使用 DI 容器管理服务依赖

**操作**:

1. **添加 NuGet 包**
   ```xml
   <PackageReference Include="Microsoft.Extensions.DependencyInjection" Version="9.0.0" />
   <PackageReference Include="Microsoft.Extensions.Configuration" Version="9.0.0" />
   <PackageReference Include="Microsoft.Extensions.Configuration.Json" Version="9.0.0" />
   ```

2. **配置服务容器**
   - 在 `App.xaml.cs` 中配置
   - 注册所有服务
   - 注册 ViewModel

3. **更新 MainWindow**
   - 通过构造函数注入 ViewModel
   - 移除直接实例化

---

### 步骤 5: 拆分 ViewModel（优先级：中）

**目标**: 将 `MainViewModel` 拆分为多个专门的 ViewModel

**操作**:

1. **创建 BaseViewModel**
   - 通用属性和命令
   - INotifyPropertyChanged 实现

2. **创建子 ViewModel**
   - ReportHeaderViewModel
   - UterusViewModel
   - EndometriumViewModel
   - AdnexaViewModel
   - PreviewViewModel

3. **MainViewModel 作为协调者**
   - 持有子 ViewModel 引用
   - 协调子 ViewModel 之间的交互

---

### 步骤 6: 性能优化（优先级：中）

**目标**: 优化预览刷新和事件订阅

**操作**:

1. **实现预览刷新节流**
   - 使用 Timer 延迟执行
   - 300ms 节流间隔

2. **优化事件订阅**
   - 使用 WeakEventManager
   - 统一管理订阅生命周期

3. **异步操作优化**
   - 所有耗时操作使用 async/await
   - 避免阻塞 UI 线程

---

### 步骤 7: 添加验证框架（优先级：低）

**目标**: 实现数据验证

**操作**:

1. **实现 ReportValidator**
   - 验证必填字段
   - 验证数值范围
   - 验证业务规则

2. **在 ViewModel 中集成**
   - 实时验证
   - 显示验证错误

---

### 步骤 8: 添加测试（优先级：低）

**目标**: 编写单元测试

**操作**:

1. **创建测试项目**
   ```bash
   dotnet new xunit -n SURS.Tests
   ```

2. **编写测试**
   - ORadsCalculator 测试
   - ReportValidator 测试
   - ViewModel 测试（使用 Mock）

## 🔧 关键技术决策

### 1. 依赖注入容器选择

**推荐**: Microsoft.Extensions.DependencyInjection

**理由**:
- .NET 官方支持
- 轻量级，性能好
- 与配置系统集成良好

### 2. 数据持久化方案

**当前阶段**: JSON 文件存储

**未来扩展**: SQLite 数据库（当需要复杂查询时）

### 3. 日志框架

**推荐**: 自定义简单实现（当前）

**未来扩展**: Serilog 或 NLog（当需要更强大功能时）

### 4. 验证框架

**推荐**: 自定义验证器（当前）

**未来扩展**: FluentValidation（当验证规则复杂时）

## ❓ 常见问题

### Q: 是否需要立即拆分所有模块？

**A**: 不需要。建议分阶段实施，优先完成核心架构（Core + Infrastructure），然后再优化 App 层。

### Q: 如何保证重构不影响现有功能？

**A**: 
1. 保持接口兼容
2. 逐步迁移，每次迁移后测试
3. 使用 Git 分支管理

### Q: 是否需要引入第三方框架？

**A**: 当前阶段不需要。使用 .NET 官方库即可。未来可根据需要引入（如 AutoMapper、FluentValidation 等）。
