## 一、架构概述

### 1.1 架构原则

- **单一职责原则 (SRP)**: 每个类/模块只负责一个功能
- **依赖倒置原则 (DIP)**: 依赖抽象而非具体实现
- **开闭原则 (OCP)**: 对扩展开放，对修改关闭
- **接口隔离原则 (ISP)**: 使用多个专用接口而非单一通用接口
- **关注点分离**: UI、业务逻辑、数据访问分离

### 1.2 目标架构

采用 **分层架构 + 模块化设计**，支持：
- ✅ 高内聚、低耦合
- ✅ 易于测试和维护
- ✅ 支持未来扩展（多窗口、Web API、移动端）
- ✅ 代码复用性强

## 四、依赖注入设计

### 4.1 推荐使用 Microsoft.Extensions.DependencyInjection

**优势**:
- .NET 官方支持
- 轻量级，性能好
- 与 .NET 生态集成良好

### 4.2 依赖注入配置

```csharp
// SURS.App/App.xaml.cs
public partial class App : Application
{
    private ServiceProvider? _serviceProvider;

    protected override void OnStartup(StartupEventArgs e)
    {
        base.OnStartup(e);

        var services = new ServiceCollection();
        ConfigureServices(services);
        _serviceProvider = services.BuildServiceProvider();

        var mainWindow = _serviceProvider.GetRequiredService<MainWindow>();
        mainWindow.Show();
    }

    private void ConfigureServices(IServiceCollection services)
    {
        // Core Services
        services.AddSingleton<IORadsCalculator, ORadsCalculator>();
        services.AddSingleton<IReportValidator, ReportValidator>();
        services.AddSingleton<IReportTextGenerator, ReportTextGenerator>();

        // Infrastructure Services
        services.AddSingleton<IPdfGenerator, PdfService>();
        services.AddSingleton<IReportRepository, FileReportRepository>();
        services.AddSingleton<IFileStorage, LocalFileStorage>();
        services.AddSingleton<IConfigurationService, ConfigurationService>();
        services.AddSingleton<ILogger, FileLogger>();

        // App Services
        services.AddSingleton<IDialogService, DialogService>();

        // ViewModels
        services.AddTransient<MainViewModel>();
        services.AddTransient<ReportHeaderViewModel>();
        services.AddTransient<UterusViewModel>();
        // ... 其他 ViewModels

        // Views
        services.AddTransient<MainWindow>();
    }

    protected override void OnExit(ExitEventArgs e)
    {
        _serviceProvider?.Dispose();
        base.OnExit(e);
    }
}
```

## 五、数据持久化方案

### 5.1 当前问题
- ❌ 没有数据持久化
- ❌ 用户数据无法保存和恢复
- ❌ 无法查看历史报告

### 5.2 推荐方案

#### 方案 A: JSON 文件存储（简单、快速实现）

**优点**:
- 实现简单
- 无需数据库
- 文件可读性强
- 适合单机应用

**实现**:
```csharp
// SURS.Infrastructure/Persistence/Repositories/FileReportRepository.cs
public class FileReportRepository : IReportRepository
{
    private readonly IFileStorage _fileStorage;
    private readonly string _reportsDirectory;

    public async Task SaveAsync(SursReport report, string filePath)
    {
        var json = JsonSerializer.Serialize(report, new JsonSerializerOptions 
        { 
            WriteIndented = true 
        });
        await _fileStorage.WriteAllTextAsync(filePath, json);
    }

    public async Task<SursReport?> LoadAsync(string filePath)
    {
        var json = await _fileStorage.ReadAllTextAsync(filePath);
        return JsonSerializer.Deserialize<SursReport>(json);
    }
}
```

#### 方案 B: SQLite 数据库（未来扩展）

**优点**:
- 支持复杂查询
- 支持多用户
- 数据完整性更好
- 可扩展为客户端-服务器架构

**实施时机**: 当需要多用户、历史记录查询、统计分析时

## 八、验证框架

### 8.1 验证需求

- 必填字段验证
- 数值范围验证（尺寸、年龄）
- 业务规则验证（日期合理性）
- 数据完整性验证

### 8.2 实现方案

```csharp
// SURS.Core/Business/ReportValidator.cs
public class ReportValidator : IReportValidator
{
    public ValidationResult Validate(SursReport report)
    {
        var errors = new List<ValidationError>();

        // 验证患者信息
        if (string.IsNullOrWhiteSpace(report.PatientName))
            errors.Add(new ValidationError("PatientName", "患者姓名不能为空"));

        // 验证年龄
        if (!int.TryParse(report.Age, out int age) || age < 0 || age > 150)
            errors.Add(new ValidationError("Age", "年龄必须在 0-150 之间"));

        // 验证日期
        if (report.LmpYear.HasValue && report.LmpMonth.HasValue && report.LmpDay.HasValue)
        {
            try
            {
                var lmpDate = new DateTime(report.LmpYear.Value, report.LmpMonth.Value, report.LmpDay.Value);
                if (lmpDate > DateTime.Now)
                    errors.Add(new ValidationError("LastMenstrualPeriod", "末次月经日期不能晚于今天"));
            }
            catch
            {
                errors.Add(new ValidationError("LastMenstrualPeriod", "末次月经日期无效"));
            }
        }

        return new ValidationResult(errors);
    }
}
```

## 九、性能优化

### 9.1 当前性能问题

1. **预览刷新无节流**: 每次输入都触发 PDF 生成
2. **事件订阅可能泄漏**: 嵌套对象订阅管理不当
3. **同步操作阻塞 UI**: PDF 生成在主线程

### 9.2 优化方案

#### 9.2.1 预览刷新节流

```csharp
// 使用 Reactive Extensions 或自定义节流
private readonly System.Threading.Timer _previewThrottleTimer;
private const int PreviewThrottleMs = 300; // 300ms 节流

private void TriggerPreviewUpdate()
{
    _previewThrottleTimer?.Change(PreviewThrottleMs, Timeout.Infinite);
}

private void OnPreviewThrottleElapsed(object? state)
{
    // 实际执行预览更新
    UpdatePreviewAsync();
}
```

#### 9.2.2 异步操作优化

```csharp
// 所有耗时操作使用 async/await
public async Task ExportPdfAsync(string filePath)
{
    IsExporting = true;
    try
    {
        await Task.Run(() => _pdfService.GeneratePdf(Report, filePath));
        await _dialogService.ShowMessageAsync("导出成功", "PDF 报告已生成");
    }
    catch (Exception ex)
    {
        _logger.LogError("导出 PDF 失败", ex);
        await _dialogService.ShowErrorAsync("导出失败", ex.Message);
    }
    finally
    {
        IsExporting = false;
    }
}
```

#### 9.2.3 内存优化

- 使用 `WeakEventManager` 管理事件订阅
- 及时释放大对象（BitmapImage）
- 使用对象池复用临时对象
